# Strong Opinions, Weakly Held (Research Protocol)

> "The ability to change your mind in the face of new evidence is a superpower."

## The Core Algorithm for Truth

1.  **Form a Hypothesis (The Strong Opinion)**
    *   Do not start with "What is X?". Start with "I believe X is true because Y."
    *   *Example*: "I believe server-side rendering is unnecessary for this app."
    *   This gives you a specific vector to attack.

2.  **Attack the Hypothesis (The Disconfirmation)**
    *   Spend 80% of your time trying to prove yourself **WRONG**.
    *   Search for "Why SSR is critical", "SPA SEO failures", "Client-side perf issues".
    *   If you only search for confirmation, you are not researching; you are masturbating.

3.  **The Pivot (The Weak Hold)**
    *   If you find 3 verified data points contradicting your view, **FLIP IMMEDIATELY**.
    *   Do not defend the ego. Defend the Truth.

4.  **Synthesis (The Output)**
    *   Your final report should say: "I initially thought X, but evidence A, B, and C proved Y is actually the case."
