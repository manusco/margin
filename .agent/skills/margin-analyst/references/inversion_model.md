# The Inversion Model (avoiding Stupidity)

> "It is remarkable how much long-term advantage people like us have gotten by trying to be consistently not stupid, instead of trying to be very intelligent." - Charlie Munger

## How to use Inversion

Instead of asking "How do we succeed?", ask **"How do we guarantee failure?"**

**Goal**: A successful Content Deck.

**Inversion (Failure Modes)**:
1.  Make it boring (Wall of text).
2.  Make it confusing (Jargon).
3.  Make it generic (Stock photos, standard layout).
4.  Make it irrelevant (Solve a problem they don't have).

## The Checklist
To win, we simply ensure we **DO NOT** do steps 1-4.
*   [ ] Did we remove all walls of text?
*   [ ] Did we kill all jargon?
*   [ ] Did we use custom visuals?
*   [ ] Is the problem painful?
